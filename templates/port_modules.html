EchoLink allocation
EchoLink can only be enabled on one configured port in this build. This is because a normal EchoLink instance uses one callsign/login and one network service from this device/IP address.

Choose the port that should own EchoLink, or select None if this build will not use EchoLink.

This choice is part of the dashboard configuration and may be changed later through the rebuild/reconfigure workflow when that function is available.
